# 기초 백엔드 스터디 4주차

## 1. 데이터베이스의 개념과 종류
데이터베이스는 전자적으로 저장되고 체계적인 데이터 모음입니다. 예를 들어, 홍길동이라는 사람의 정보를 체계적으로 작성하면 다음과 같습니다:

- 홍길동 010-xxxx-xxxx 주소: 서울특별시 마포구...

이것은 데이터입니다.

이러한 데이터들의 모음:

- 홍길동 010-xxxx-xxxx 주소: 서울특별시 마포구...
- 김닐동 010-oooo-oooo 주소: 서울특별시 금천구...
- 박철동 010-vvvv-vvvv 주소: 서울특별시 양천구...

이렇게 표현된 데이터들의 모음을 데이터베이스라고 합니다.

타 학회에서 전처리와 관련된 학습을 하고 있는데, 이 때 `train.csv`라는 타이타닉의 데이터베이스를 사용합니다. 이 또한 데이터베이스라고 할 수 있습니다.

엑셀로 사용자의 개인정보를 저장하고 정리한다면, 이것도 데이터베이스로 볼 수 있습니다.

## 2. RDBMS의 특성
RDBMS는 "관계형 데이터베이스를 생성, 업데이트, 관리하는 데 사용하는 프로그램"입니다. MySQL, PostgreSQL 등이 그 예입니다.

위의 데이터베이스를 예로 들어 보겠습니다:

- 홍길동 010-xxxx-xxxx 주소: 서울특별시 마포구...
- 김닐동 010-oooo-oooo 주소: 서울특별시 금천구...
- 박철동 010-vvvv-vvvv 주소: 서울특별시 양천구...
- 박철동 010-aaaa-aaaa 주소: 부산광역시 해운대구...

이 경우 서로 다른 박철동씨를 두 명이나 갖게 됩니다. 데이터베이스에 "박철동씨의 정보를 줘!" 했을 때, 필요한 박철동씨를 못 찾을 수도 있습니다.

이를 구분하기 위해 각 데이터 앞에 태그를 붙여 보겠습니다:

- 1. 홍길동 010-xxxx-xxxx 주소: 서울특별시 마포구...
- 2. 김닐동 010-oooo-oooo 주소: 서울특별시 금천구...
- 3. 박철동 010-vvvv-vvvv 주소: 서울특별시 양천구...
- 4. 박철동 010-aaaa-aaaa 주소: 부산광역시 해운대구...

이렇게 하면 "4번 사람의 데이터를 요청"할 때 중복 없이 처리됩니다. 이러한 번호를 ID라고 하며, 다른 테이블에서 ID를 통해 고객의 데이터를 중복 없이 손쉽게 가져올 수 있습니다. 

이처럼 상호작용하는 서로 다른 테이블을 관계형 데이터베이스(RDB)라고 하며, 이를 관리하는 시스템이 관계형 데이터베이스 관리 시스템(RDBMS)입니다.

## 3. 장고 Model과 사용법
실습을 통해 게시판을 만들면서 Django의 모델 사용법을 확인할 수 있었습니다. `models.py` 파일을 확인해보면 다음과 같습니다:

```py
from django.db import models

# Create your models here.
class Question(models.Model):
    subject = models.CharField(max_length=200)
    context = models.TextField()
    creat_date = models.DateTimeField()
```
위의 코드에서:

subject는 CharField 함수를 사용해 200자까지 문자를 저장합니다.
context는 TextField 함수를 사용해 길이가 정해지지 않은 문자열을 저장합니다.
creat_date는 DateTimeField 함수를 사용해 날짜와 시간을 저장합니다.
추가적으로, auto_now_add 옵션을 통해 데이터 생성 시 현재 시간을 저장하도록 지정할 수 있습니다. 이 경우의 코드는 다음과 같습니다:

```py
class Question(models.Model):
    creat_date = models.DateTimeField(auto_now_add=True)
```
실습에서 했던 마이그레이션 과정을 통해 모델을 반영하고, 웹을 통해 확인할 수 있었습니다.
